# CONVENTIONS

## Naming
- Classes: PascalCase
- Methods: camelCase
- Constants: UPPER_SNAKE_CASE
- Variables: snake_case

## Patterns
- Inyección de dependencias para servicios
- Manejo de errores con validación determinista
- Testing con casos de usuario (no solo unitarios)

## Commit format
- Conventional commits: feat:, fix:, chore:, docs:, refactor:, test:
- Emojis opcionales para mayor legibilidad

## What to avoid
- Inventar herramientas o APIs no existentes
- Confiar ciegamente en las respuestas del LLM
- Mezclar verdad del sistema con prompts del sistema
- Usar SentenceTransformers (usar TF-IDF manual)
